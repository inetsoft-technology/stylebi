Table Structure Tokens (Phase 5 Part C)

Server-rendered viewsheet table structure: gridlines, header hierarchy, and grand-total emphasis. Applied as gated modern defaults by overlaying a per-assembly clone of the shipped "Default Style" in DataVSAQuery, so the live view and every export format agree and user cell formats still win. Values map the example mockups' relationships (subtle low-contrast gridlines; a quiet header rather than a heavy fill) into the warm-neutral visualization system rather than copying the mockups' purple/blue theme. Two border tiers: a subtle interior gridline (#E8E5DE) and a stronger header→body rule (#D9D5CC), which equals $shell-border-default so the viewsheet header rule matches shell/DOM tables. Fill hierarchy: header #F1EFEA, subtotal #EEEAE1, grand total #E9E4DA (each deeper). Subtotal (#EEEAE1) and grand-total (#E9E4DA) are proposed values with no design spec yet. Grand total is emitted via the positional trailer band; group-subtotal emphasis is deferred (item 3b) and not emitted yet. Dark-mode variants are reference only — deferred, not emitted yet.

Chart Palette Comparison

This layout puts the current default palette next to two possible future alternatives for faster comparison. The default palette reflects the current shared chart system. The other two are exploratory only: one pushes toward a more adventurous non-standard BI feel, and one tests repeated hue families such as light blue and deep blue appearing earlier in the sequence.

Default Palette - Light
Phase 8 gated modern categorical default (series 1-8). Indices 9-40 keep the legacy COLOR_PALETTE tail. Gate-off renders the legacy palette unchanged.
